DATA FLOW EXAMPLE
=================

User wants to change cat's background to a living room:

1. User clicks on cat in frontend
   ↓
2. Frontend sends click to backend /editing/apply_mask
   ↓
3. Backend runs MobileSAM (local or our machine backend) → generates cat mask
   ↓
4. Frontend displays mask preview to user
   ↓
5. User enters prompt: "cozy living room with plants"
   ↓
6. Frontend sends to backend /editing/qwen_edit
   ↓
7. Backend forwards to Modal (Qwen white-to-scene endpoint)
   Request: {
     image_base64: "...",
     mask_base64: "...",
     prompt: "cozy living room with plants"
   }
   ↓
8. Modal runs Qwen inference (~5-10 seconds)
   ↓
9. Modal returns edited image
   ↓
10. Backend forwards to frontend
    ↓
11. Frontend displays result to user

COST CALCULATION EXAMPLE
=========================

Scenario: 100 image edits per day

Assumptions:
- 50% use Qwen (5s each)
- 50% use SDXL (5s each)
- Container stays warm 2 hours/day

Qwen:
  50 edits × 5s = 250s = 0.07 hours
  0.07 × $2.50 = $0.175/day

SDXL:
  50 edits × 5s = 250s = 0.07 hours
  0.07 × $0.50 = $0.035/day

Warm time:
  Qwen: 2 hours × $2.50 = $5/day
  SDXL: 2 hours × $0.50 = $1/day

Total: ~$6.21/day = ~$186/month

With free $30 credit: First ~5 days free!

To reduce costs:
- Lower container_idle_timeout (containers shut down faster)
- Use A10G for both (slower but cheaper)
- Batch requests (process multiple at once)

NEXT STEPS
==========

1. Install Modal:
   pip install modal

2. Authenticate:
   modal token new

3. Deploy servers:
   modal deploy modal_qwen_server.py
   modal deploy modal_sdxl_server.py

4. Get your URLs from Modal output

5. Update test_modal_client.py with your URLs

6. Test:
   python test_modal_client.py

7. Integrate with your FastAPI backend:
   - Add endpoints in Routers/segmentation_inpainting.py
   - Call Modal URLs with requests library
   - See test_modal_client.py for examples

8. Update frontend to call new backend endpoints

9. Done!
